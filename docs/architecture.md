# 系统架构

## 1. 系统边界

本项目面向双级轴流式燃气调压器及其试验台。现场数据通过智能开度指示器/边缘采控终端汇聚，再经5G网络上传。

## 2. 分层架构

```text
传感器、调压器、试验台执行机构
        ↓
智能开度指示器 / 边缘采控终端
        ↓  MQTT over 5G
ThingsBoard CE
        ↓
分析服务 / 预测服务 / 报告服务
        ↓
Vue 3 + Three.js 专用数字孪生界面
```

## 3. 现场层

- 一级进口压力变送器
- 中间级压力变送器
- 二级出口压力变送器
- 一级和二级位移传感器
- 燃气流量计
- 温度传感器
- 切断阀和放散阀状态
- PLC及本地安全回路

## 4. 平台层

ThingsBoard CE负责：

- 设备注册和鉴权
- MQTT遥测接入
- 时序数据存储
- 基础看板
- 规则链和告警
- RPC控制消息
- 用户和权限

## 5. 自研服务

### Analytics Service

- 调压精度计算
- 稳态误差和波动分析
- 超调量、响应时间和调节时间
- 一级、二级压降分析
- 双级协调性分析
- 关闭性能分析

### Prediction Service

- 出口压力短时预测
- 中间级压力预测
- 流量和流速趋势预测
- 极限开度风险预测
- 双级失配和异常风险预测

### Twin UI

- 实时工艺流程图
- 压力、流量和阀位曲线
- 双级阀位动画
- 告警与预测结果
- 试验任务和报告

## 6. 安全边界

云端平台只负责监测、试验编排、参数建议和受控指令下发。紧急切断、泄漏联锁、超压保护和通信失效保护必须在本地PLC或独立安全回路执行。
