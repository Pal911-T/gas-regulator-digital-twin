# UI 基线决策：以“睿轴驭燃 v3.3”继续优化

## 决策

停止继续扩展当前简化版 Vue 大屏。后续前端以用户现有的 `睿轴驭燃_v3.3_修复完整版` 为产品与视觉基线，并在 GitHub 仓库中持续版本化改造。

这不是放弃 GitHub 方案，而是把 GitHub 从“另起炉灶开发简化界面”调整为“管理、测试、发布和持续优化 v3.3 成熟原型”。

## 原因

v3.3 已具备更成熟的：

- 工业视觉体系与信息层级；
- 系统总览、实时监控、阀位监测、压力分析、流量分析、设备管理；
- 多设备卡片、趋势图、仪表、告警日志和统计；
- 运行态势、健康度、同步状态、风险评估；
- 参数配置、维护记录、连接状态和工况模拟；
- 模型结果、残差、安全裕度和处置建议。

当前 Vue 原型只保留为早期技术试验，不再作为主产品路线。

## 后续升级目标

在 v3.3 基础上统一替换为 RTJ-25/4.0Z 双级轴流调压器业务模型：

- 单设备双级结构，而不是三台普通调压器并列；
- P1 进口压力、Pm 中间压力、P2 出口压力；
- X1/X2 位移与一级/二级开度；
- 标准流量 Qn、温度 T、设定压力；
- 切断、放散、设备、通信和数据质量状态；
- 智能阀位指示器/边缘采集终端；
- 外接压力、流量、温度传感器和 PLC 开关量；
- 5G、MQTT、时序数据库和分析服务的逻辑链路展示；
- 数据明确区分仿真值、设定值、计算值、预测值和未来测量值；
- 精度试验、告警处置、工况注入、历史记录和报告页面。

## 页面迁移顺序

1. 保留 v3.3 的总体视觉、顶部导航、卡片、图表和弹窗体系。
2. 将系统总览改造成 RTJ-25/4.0Z 单设备双级数字孪生总览。
3. 将实时监控扩展为 P1/Pm/P2、X1/X2、Qn、T 和状态量。
4. 将阀位监测升级为一级、二级阀位响应和卡涩工况。
5. 将压力分析升级为三级压力链、设定值偏差和 AC5 评价。
6. 将流量分析升级为流量—开度—压力关联分析。
7. 在设备管理中加入采集通道、5G/MQTT、阈值和数据来源说明。
8. 新增采集链路、告警中心、精度试验、预测预警和操作日志。

## 当前阶段边界

当前交付是功能原型：

- 展示未来真实采集与通信逻辑；
- 使用设计参数、公开资料和典型工况生成仿真数据；
- 暂不实现真实 5G、MQTT、数据库、传感器或 PLC 接入；
- 不允许把仿真值标记为现场测量值；
- 云端页面不承担最终安全联锁职责。
